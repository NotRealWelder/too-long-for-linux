# too-long-for-linux.py

*Простой скрипт для проверки длинных путей и имен файлов в Linux с учетом системных ограничений
(4096 символов для пути, 255 для имени)*

---

### Опции


| *Опция*                    | *Описание*                 |
|----------------------------|----------------------------|
| **-h**, **--help**         | Показать справку и выйти   |
| **-p**, **--no-progress**  | Скрыть индикатор прогресса |
| **-q**, **--quiet**        | Тихий режим (только цифры) |
| **-l FILE**, **--log FILE**| Сохранить результаты в файл|
| **--axe**                  | Осторожно! Автоматически исправить длинные имена|

---

### Примеры использования

```bash
# Обычная проверка
./too-long-for-linux.py /mnt/tmp

# Проверка без прогресс-бара + сохранение в файл
./too-long-for-linux.py -p -l results.txt /mnt/tmp

# Только статистика (тихий режим)
./too-long-for-linux.py -q /mnt/tmp

# Авто-исправление (используйте с осторожностью!)
sudo ./too-long-for-linux.py --axe /mnt/tmp
```
---

### Особенности:

- Скрипт проверяет имена и пути файлов и директорий с учётом ограничения Linux (255 символов для имени, 4096 для полного пути)
- Работает рекурсивно в текущей или указанной директории
- Возможность сохранения LOG-файла
- Для автоматического исправления могут понадобиться права root

---

### Зачем это нужно?

Иногда при переносе файлов из Windows в Linux возникают проблемы с длинными именами файлов. Этот скрипт помогает найти и исправить такие проблемы.
